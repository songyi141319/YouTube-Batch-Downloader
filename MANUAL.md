## 1. 系统要求

| 项目 | 要求 |
|------|------|
| 操作系统 | Windows 10+ / macOS 12+ |
| Python | 3.10 或更高版本 |
| FFmpeg | 需安装并加入系统 PATH |
| 网络 | 需要能访问 YouTube |
| 磁盘空间 | 视下载内容而定，建议预留 10GB+ |

### 检查 Python 版本

```bash
python3 --version   # macOS
python --version     # Windows
```

### 检查 FFmpeg

```bash
ffmpeg -version
```

如果未安装 FFmpeg：
- **Windows**：从 https://ffmpeg.org/download.html 下载，解压后将 bin 目录加入系统 PATH
- **macOS**：`brew install ffmpeg`

## 2. 安装配置

### 方式一：从 GitHub 下载源码

```bash
git clone https://github.com/songyi141319/YouTube-Batch-Downloader.git
cd YouTube-Batch-Downloader
pip install -r requirements.txt
```

### 方式二：下载 Release 压缩包

从 [Releases 页面](https://github.com/songyi141319/YouTube-Batch-Downloader/releases) 下载对应平台的 zip 文件，解压即可使用。

### 依赖说明

本工具仅依赖一个第三方库：

- **yt-dlp**：YouTube 视频下载核心库

首次运行时，程序会自动检测 yt-dlp 是否已安装。如未安装，会自动通过 pip 安装。

## 3. 界面介绍

程序采用深靛蓝（#1A1A2E）+ 琥珀金（#E2B714）配色方案。

### 3.1 顶栏

- 左侧：程序标题
- 右侧：「打赏支持」按钮
- 金色装饰线分隔

### 3.2 链接输入区

白色卡片样式的多行文本框，用于输入 YouTube 播放列表链接。每行一个链接。

### 3.3 操作按钮行

| 按钮 | 功能 | 颜色 |
|------|------|------|
| 解析合集 | 解析输入的所有播放列表链接 | 深靛蓝 |
| 挑选视频 | 解析后，逐个合集选择要下载的视频 | 琥珀金 |
| 直接下载 | 跳过解析和挑选，直接下载全部 | 灰色 |
| 开始批量下载 | 按选择结果开始下载 | 绿色 |
| 清空 | 清空所有输入和列表 | 灰色 |
| 粘贴 | 从剪贴板粘贴链接 | 灰色 |

### 3.4 保存位置

显示当前下载保存路径。点击「更改」按钮可选择新的保存目录。

默认保存位置：`~/Downloads/YouTube视频合集/`

勾选「同时下载字幕」后，会在视频同目录下的「字幕」子文件夹保存 SRT 格式字幕文件。

支持的字幕语言：
- 中文简体 (zh-Hans)
- 中文繁体 (zh-Hant)
- 英文 (en)

### 3.6 合集列表

显示已成功解析的播放列表及其视频数量。

### 3.7 下载日志

深色终端风格的日志窗口，实时显示下载过程：

| 颜色 | 含义 |
|------|------|
| 青色 | 一般信息 (INFO) |
| 灰色 | 详细信息 (DETAIL) |
| 绿色 | 成功 (SUCCESS) |
| 黄色 | 警告 (WARN) |

琥珀金色进度条，显示整体下载进度。上方文字显示当前合集/视频/总进度。

- **重试失败**：重新下载所有失败的视频

## 4. 功能详解

### 4.1 解析播放列表

点击「解析合集」按钮后，程序会：

1. 检测 yt-dlp 是否已安装
2. 逐个解析输入的播放列表链接
3. 提取每个播放列表的标题和视频列表
4. 在合集列表中显示解析结果

解析超时时间为 30 秒。如果播放列表很大，可能需要多次尝试。

### 4.2 视频选择

点击「挑选视频」按钮后，会依次弹出每个播放列表的选择对话框：

- 默认全选所有视频
- 可以逐个勾选/取消
- 支持鼠标拖拽批量勾选
- 快速操作：全选、取消全选、反选、前10个、前20个
- 确认后更新合集列表中的视频数量

### 4.3 直接下载模式

点击「直接下载」按钮，跳过解析和挑选步骤，直接下载输入链接中的所有视频。适合不需要挑选的场景。

### 4.4 批量下载模式

点击「开始批量下载」按钮，按以下流程下载：

1. 为每个播放列表创建独立文件夹
2. 按视频编号顺序下载
3. 自动选择最高画质
4. 自动合并视频和音频为 MP4
5. 文件名格式：`01_视频标题.mp4`

## 5. 下载选项

### 5.1 视频格式

- 优先选择：MP4 视频 + M4A 音频
- 备选方案：最佳视频 + 最佳音频
- 最终兜底：best（单一最佳格式）
- 输出格式：统一合并为 MP4

### 5.2 文件命名

```
保存目录/
  ├── 播放列表标题1/
  │   ├── 01_视频标题1.mp4
  │   ├── 02_视频标题2.mp4
  │   ├── 字幕/          （启用字幕时）
  │   │   ├── 01_视频标题1.zh-Hans.srt
  │   │   └── 01_视频标题1.en.srt
  │   └── .download_progress.json
  └── 播放列表标题2/
      └── ...
```

## 7. 断点续传机制

每个播放列表文件夹下会生成 `.download_progress.json` 文件，记录每个视频的下载状态：

- 已完成的视频在重新下载时自动跳过
- 中断后重新运行，已下载的视频不会重复下载
- 可手动删除该文件来强制重新下载所有视频

## 8. 常见问题

### Q: 下载速度慢？
**A:** 下载速度取决于网络环境和 YouTube 服务器。确保网络连接稳定。

### Q: 提示 FFmpeg 未找到？
**A:** 请安装 FFmpeg 并确保 `ffmpeg` 命令可在终端中直接运行。

### Q: 解析超时？
**A:** 大型播放列表（100+ 视频）可能需要更长时间。如果反复超时，请检查网络。

### Q: 下载失败？
**A:** 可能原因：
- 视频已被删除或设为私有
- 地区限制
- 网络连接中断
- yt-dlp 版本过旧（尝试 `pip install --upgrade yt-dlp`）

### Q: 如何更新 yt-dlp？
**A:** 运行 `pip install --upgrade yt-dlp`

### Q: macOS 首次运行提示"无法验证开发者"？
**A:** 右键点击 `启动下载工具.command` → 打开 → 确认打开

## 9. 技术细节

### 9.1 核心依赖

| 库 | 用途 |
|------|------|
| yt-dlp | YouTube 视频信息提取和下载 |
| tkinter | 图形界面（Python 内置） |
| FFmpeg | 视频/音频格式合并（外部工具） |

### 9.2 下载命令参数

```
yt-dlp
  -f bestvideo[ext=mp4]+bestaudio[ext=m4a]/bestvideo+bestaudio/best
  --merge-output-format mp4
  -o "保存路径/序号_标题.扩展名"
  --no-playlist
  --newline
```

### 9.3 字幕命令参数

```
--write-sub --write-auto-sub
--sub-lang zh-Hans,zh-Hant,en
--convert-subs srt
```

### 9.4 跨平台兼容

- **字体**：macOS 使用 PingFang SC / SF Mono，Windows 使用 Microsoft YaHei UI / Consolas
- **Python 路径**：自动检测系统 Python
- **保存路径**：默认使用用户 Downloads 目录
